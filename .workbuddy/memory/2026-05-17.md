## BugFix：Enka 导入 CORS + 副词条按钮无反应

### Bug 1: Enka 导入网络请求失败
- **根因**：浏览器直接请求 `https://enka.network/api/uid/{uid}`，被 CORS 拦截
- **修复**：
  - `vite.config.ts` 添加 `server.proxy` 配置，`/enka-api` 代理到 `https://enka.network`
  - `src/services/enka.ts` 的 `ENKA_API_BASE` 改为 `'/enka-api/api/uid'`（相对路径走代理）

### Bug 2: 手动添加圣遗物副词条按钮无反应
- **根因**：`ArtifactSlotEditor` 中 `handleAddSubStat` 第一行 `if (!artifact) return`，但 artifact 初始为 null 时按钮仍然渲染（条件 `artifact?.subStats.length ?? 0 < 4` 在 null 时为 true）
- **修复**：`src/components/artifact/ArtifactEditor.tsx` 中按钮渲染条件改为 `artifact != null && artifact.subStats.length < 4`，null 时显示"请先选择主词条"提示
