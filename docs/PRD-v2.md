# 原神圣遗物词条优化器 V2 — 增量产品需求文档（PRD）

## 项目信息

| 字段 | 值 |
|---|---|
| 项目名称 | `genshin_artifact_optimizer_v2` |
| 语言 | 中文 |
| 技术栈 | Vite + React + MUI + Tailwind CSS + TypeScript + Zustand |
| 基线版本 | V1（当前 master 分支） |
| 原始需求 | 将「词条重优化」与「理想模板」两个独立页面合并为统一角色分析工作流；新增武器/命座录入、伤害前后对比、乘区词条分析、角色存档系统 |

---

## 一、产品目标（V2 核心价值主张）

1. **一站式角色分析闭环**：将「录入 → 计算 → 优化 → 存档」串成连贯工作流，用户无需在两页面间跳转即可完成角色圣遗物的完整评估与优化
2. **伤害可量化可对比**：在优化前后各做一次伤害计算，用具体数值让用户直观感受到「换词条到底提升了多少」，而非仅看抽象的词条分配差异
3. **低门槛高自由度**：武器被动与命座由用户按乘区手动填入——不做数据库，降低工具维护成本；同时给予高级玩家精确控制每个加成数值的自由度

---

## 二、用户故事

1. **As a** 刚抽到新角色的玩家，**I want to** 一站式录入角色+武器+命座+圣遗物数据，**so that** 我能立刻看到当前配置的伤害与优化空间
2. **As a** 想提升输出的玩家，**I want to** 看到优化前后的伤害数值对比，**so that** 我能直观判断词条调整是否值得
3. **As a** 数据党玩家，**I want to** 按乘区查看词条分配变化及其对伤害的贡献，**so that** 我能理解「为什么暴击词条比攻击词条更值」
4. **As a** 有多把武器/多命座的玩家，**I want to** 下拉选武器自动填白值、手动填命座加成数值，**so that** 我能快速测试不同配置方案
5. **As a** 长期使用该工具的玩家，**I want to** 保存/导出/导入角色配置，**so that** 我不用每次重新录入数据
6. **As a** 使用 Enka 导入的玩家，**I want to** 导出时选择是否包含 UID，**so that** 我分享配置时不泄露个人信息

## 三、功能需求池

### P0 — Must Have

| 编号 | 需求 | 说明 |
|---|---|---|
| P0-01 | 统一角色分析页面 | 合并 RedistributePage + IdealTemplatePage 为单页面，Tab 切换三个步骤：角色与装备 → 伤害计算 → 词条分析与方案 |
| P0-02 | 武器选择与录入 | 下拉选武器名 → 自动填入基础攻击力 + 副词条类型及数值；武器被动效果由用户手动填写各乘区数值 |
| P0-03 | 命座加成录入 | 用户手动填写各命座在各乘区的数值加成，按乘区分组展示输入框（攻击区/增伤区/暴击区/精通区/防御区/抗性区/基础伤害区） |
| P0-04 | 伤害前后对比 | 基于预设典型场景，计算当前配置（未优化）与优化后配置的伤害，并展示差值与提升百分比 |
| P0-05 | 典型场景预设 | 每个角色预设 2-3 个典型伤害场景（如"胡桃蒸发重击"），包含：技能选择、反应类型、敌人假设（等级/抗性） |
| P0-06 | 乘区词条分析表 | 表格展示：乘区名 \| 当前词条数 \| 优化后词条数 \| 变化 \| 对伤害贡献 |
| P0-07 | 角色存档（本地） | 用户可将当前角色完整配置保存到 localStorage，支持多个存档槽位 |
| P0-08 | 角色导出/导入 JSON | 角色数据可导出为 JSON 文件下载；可上传 JSON 文件恢复角色数据；导入时校验文件格式合法性 |

**设计要点**：
- 武器下拉选择后自动填充基础攻击力和副词条，用户无需手动输入
- 武器被动/命座加成的输入框**按乘区分组**，与伤害引擎的乘区对齐
- 百分比字段标 `%` 后缀，固定值标注单位说明
- 用户填写的数值**叠加**到已有基础上（如角色基础暴击率5% + 命座暴击率4% = 9%）

## 五、新增数据模型定义

### 5.1 乘区加成输入（武器被动 + 命座共用）

```typescript
/**
 * 按乘区组织的额外加成数值。
 * 所有百分比字段存为小数（如 50% → 0.5），固定值字段存为原始数值。
 * 这些值叠加到已有基础上，不替换。
 */
export interface ZoneBonusInput {
  /** 攻击区加成（百分比，如 0.2 表示 +20%攻击力） */
  atkPercent?: number;
  /** 攻击区加成（固定值，如 300 表示 +300攻击力） */
  atkFlat?: number;
  /** 生命区加成（百分比） */
  hpPercent?: number;
  /** 生命区加成（固定值） */
  hpFlat?: number;
  /** 防御区加成（百分比） */
  defPercent?: number;
  /** 防御区加成（固定值） */
  defFlat?: number;
  /** 增伤区加成（百分比，如 0.3 表示 +30%伤害加成） */
  dmgBonus?: number;
  /** 暴击率加成（百分比，如 0.08 表示 +8%暴击率） */
  critRate?: number;
  /** 暴击伤害加成（百分比，如 0.4 表示 +40%暴击伤害） */
  critDmg?: number;
  /** 元素精通加成（固定值，如 60 表示 +60精通） */
  elementalMastery?: number;
  /** 元素充能效率加成（百分比） */
  energyRecharge?: number;
  /** 抗性区减抗（百分比，如 0.2 表示 -20%抗性） */
  resistReduction?: number;
  /** 基础伤害区加成（固定值，加入baseDamage） */
  baseDamageFlat?: number;
}
```

### 5.2 武器录入扩展

```typescript
/**
 * 用户选定的武器配置（含被动效果手动输入）。
 * 基础属性从武器数据自动填充，被动效果由用户手动填写。
 */
export interface WeaponConfig {
  /** 选中的武器数据（自动填充基础攻击力+副词条） */
  weaponData: WeaponData;
  /** 武器等级 */
  weaponLevel: number;
  /** 武器被动效果各乘区加成（用户手动填写） */
  passiveBonus: ZoneBonusInput;
}
```

### 5.3 命座录入

```typescript
/**
 * 命座配置。
 * 用户自主填写各乘区加成数值，不依赖命座数据库。
 */
export interface ConstellationConfig {
  /** 当前命座等级（0-6） */
  level: number;
  /** 各命座在各乘区的加成数值（用户手动填写） */
  bonus: ZoneBonusInput;
}
```

### 5.4 典型场景预设

```typescript
/**
 * 预设的典型伤害场景定义。
 * 每个角色 2-3 个场景，由开发者预设。

export interface TypicalScenario {
  /** 场景唯一ID */
  id: string;
  /** 场景显示名（如"蒸发重击"） */
  name: string;
  /** 使用的技能倍率 */
  skillMultiplier: number;

  /** 反应类型 */
  reactionType: ReactionType;
  /** 增幅反应倍率（1.5 或 2.0，仅 AMPLIFYING 路径有效） */
  amplifyingMultiplier: number;
  /** 敌人等级 */
  enemyLevel: number;
  /** 敌人抗性（小数） */
  enemyResistance: number;
  /** 敌人元素抗性修正（部分场景需要特殊值） */
  enemyElementResistance?: number;
  /** 场景描述（tooltip 展示） */
  description: string;
}

 * 角色典型场景配置文件。

  scenarios: TypicalScenario[];
```

### 5.7 角色存档

```typescript
  /** 是否来自 Enka 导入 */
  fromEnka: boolean;
  /** Enka UID（可选，仅 Enka 导入时存在） */
  uid?: string;
}
```

### 7.2 导入校验规则

| 校验项 | 规则 | 失败行为 |
|---|---|---|
| version 字段 | 必须存在且为字符串 | 拒绝导入，提示"文件格式不正确" |
| version 兼容性 | 主版本号 ≤ 当前版本主版本号 | 拒绝导入，提示"文件版本不兼容" |
| saves 数组 | 必须为非空数组 | 拒绝导入，提示"文件无有效角色数据" |
| characterId | 每个save必须包含且存在于角色数据库 | 跳过该角色，提示"角色 XXX 不受支持" |
| artifacts | 每件圣遗物的 slot 必须合法 | 跳过该件圣遗物 |
| uid 字段 | 若存在则保留，由用户在导出时选择是否包含 | 不影响导入 |

### 7.3 UID 隐私处理

- 导出时弹窗提供勾选框："包含 UID 信息"
- 默认**不勾选**（不包含 UID）
- 若用户取消勾选，导出的 JSON 中删除 `uid` 字段
- 导入时不做 UID 相关校验，有则保留，无则忽略
