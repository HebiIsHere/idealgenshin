# 理想原生 (Ideal Genshin) v3.0

基于完整四层伤害公式的**原神圣遗物词条优化工具**。逐区填写、问答引导、可视化计算。

## ✨ 功能

- **Enka 导入** — 输入 UID 一键导入角色展柜数据（角色/武器/圣遗物）
- **七步向导** — Enka导入 → 角色 → 武器 → 圣遗物 → 天赋&命座 → 倍率&反应 → 队伍增益
- **完整伤害引擎** — 四层大一统公式，覆盖直伤/增幅/剧变/激化/月反应全部路径
- **武器精炼数据** — 222 把武器真实 R1–R5 描述（来自 genshin-db），变化数值自动高亮
- **圣遗物套装** — 55 套圣遗物 2pc/4pc 效果自动检测与应用
- **天赋/命座模拟** — 自由填入各乘区数值，包含天赋&命座详情参考
- **倍率混合** — 支持单属性或双属性混合倍率（如 ATK 100% + EM 50%）
- **伤害计算流程** — 四层公式逐区展示，每个乘区显示中间值和计算公式
- **同词条重优化** — 固定总词条数，爬山算法重分配至最优；含面板对比、词条分布、乘区分析图表
- **理想模板** — 可调词条数，计算理论最优伤害及词条推荐
- **配置存档** — JSON 导入/导出，localStorage 持久化

## 🛠 技术栈

**前端**: Vite · React 18 · TypeScript · MUI v6 · Zustand · Recharts  
**引擎**: 自研四层伤害公式管道（15 个独立乘区）  
**优化器**: Web Worker 爬山算法 · 枚举+剪枝+重分配+理想模板  
**数据**: genshin-db（112 角色 / 224 武器 / 55 圣遗物套装）· Enka Network API

## 🚀 本地运行

```bash
npm install
npm run dev      # 开发服务器 → http://localhost:5173
npm run build    # 生产构建 → dist/
```

## 📁 项目结构

```
src/
├── engine/          # 伤害公式引擎（4层 × 15个乘区）
│   ├── formula.ts   # 大一统公式管道
│   ├── stats.ts     # 属性计算器
│   └── zones/       # 15个独立乘区实现
├── optimizer/       # 词条优化器（Worker线程）
├── components/      # React 组件
│   ├── wizard/      # 向导导航（滚筒+步进器）
│   ├── optimizer/   # 伤害流程/乘区分析/场景选择
│   ├── character/   # 角色面板/天赋/命座
│   ├── weapon/      # 武器选择/被动模拟
│   ├── artifact/    # 圣遗物编辑/套装选择/Enka导入
│   └── layout/      # 布局/存档管理
├── store/           # Zustand 状态管理
├── data/            # 角色/武器/圣遗物/套装/命座数据
├── services/        # Enka API / SaveService
└── pages/           # 着陆页 / 向导页
```
